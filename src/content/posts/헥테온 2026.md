# 헥테온 2026 Write-up

## plottergeist

**Category:** Forensics  
**Flag:** `hacktheon2026{the_plotter_reveals_its_secret_through_sound}`

---

## Challenge

Custom CoreXY pen plotter가 마지막 테스트 중 멈췄고, 남은 파일은 다음 두 개뿐이었다.

- `plottergeist.pcap`
- `bench_mic.wav`

PCAP에는 motion traffic만 들어있어서 어떤 이동이 실제로 잉크를 남겼는지 직접 알 수 없다. 대신 같은 세션에서 녹음된 마이크 파일에 move type별 미세한 차이가 남아 있을 수 있다는 설명이 주어졌다.

목표는 실제로 그려진 그림을 복원하고 flag를 얻는 것이다.

---

## 1. PCAP 포맷 확인

PCAP 안의 UDP payload를 보면 초기 패킷 뒤에 다음 문자열이 응답으로 온다.

```
FMT:OP|SQ|DT|AM|BM~
```

이후 대부분의 motion packet은 `0xa1` opcode를 갖고 길이가 10바이트였다. 필드는 다음처럼 big-endian으로 파싱할 수 있다.

```python
sq, dt, am, bm = struct.unpack("!HHhh", payload[1:9])
```

CoreXY 구조에서는 두 모터 이동량 `AM`, `BM`으로부터 XY 이동량을 복원할 수 있다.

```python
dx = (am + bm) // 2
dy = (am - bm) // 2
```

이를 누적하면 이동 경로가 나온다. 전체 motion packet은 940개였고, 실제 좌표는 7개의 수평 scanline으로 구성되어 있었다.

```
y = 4  : x 16..365
y = 5  : x 16..366
y = 6  : x 16..367
y = 7  : x 16..368
y = 8  : x 16..367
y = 9  : x 16..366
y = 10 : x 16..365
```

즉, plotter는 7픽셀 높이의 긴 래스터 이미지를 그리던 중이었다.

---

## 2. Move Layer 분리

수평 이동 packet을 보면 `dt=4`와 `dt=5`가 거의 번갈아 나타난다. 처음에는 단순히 `dt=5`를 ink, `dt=4`를 travel로 보면 되는 것처럼 보였지만, 이 결과는 대부분이 검게 채워진 이미지가 되어 사람이 읽기 어렵다.

반대로 `dt=4` 또는 `dt=5`의 complement를 보면 7행짜리 희소한 패턴이 나온다. 이 패턴은 일반 글자처럼 보이진 않지만, 고정 폭 7-row bitmap/symbol처럼 볼 수 있다.

### 오디오 타임스탬프 보정

오디오를 확인해 보니 PCAP timestamp와 WAV timestamp가 그대로 맞지 않았다. 전체 move activity와 오디오 envelope를 상관시켜 보면 약 `-0.139s` offset에서 가장 잘 맞았다.

```
pcap motion t = 0.140s  <->  audio t ~= 0.001s
```

따라서 오디오 feature를 계산할 때는 다음과 같이 보정했다.

```python
audio_start = motion_time - 0.1395
audio_end = audio_start + dt * 0.01
```

이 보정 후 같은 `(dt, dx, dy)` 명령끼리 비교하면 모터음 차이는 거의 제거되고, 실제 pen/contact 차이로 보이는 작은 차이를 확인할 수 있었다. 최종적으로 **희소 레이어(`dt=4`)가 실제 정보 레이어**라고 판단했다.

## 3. 7-row Raster 복원

수평 scanline별로 `dt == 4` 구간을 칠하면 다음과 같은 7행 비트맵이 된다.

```python
B = np.zeros((7, width), dtype=np.uint8)

for row_index, row in enumerate(rows):
    for i in row:
        move = motions[i]
        if move["dt"] == 4:
            x0 = move["x0"] - min_x
            x1 = move["x1"] - min_x
            B[row_index, x0:x1] = 1
```

전체 폭은 352px였다. 맨 끝 일부를 제외하면 6px 단위로 나누었을 때 prefix가 자연스럽게 맞는다.

```
352 / 6 = 58 remainder 4
→ 58개의 6-column symbol
```

## 4. Known-plaintext로 Symbol 해독

Challenge에서 알려준 prefix `hacktheon2026{`를 앞에서부터 대입하면 각 symbol의 반복이 모순 없이 맞는다.

앞 14개 symbol의 대응은 다음과 같다.

| Symbol | A | B | C | D | E | A | F | C | G | H | I | H | J | K |
|--------|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| Char   | h | a | c | k | t | h | e | o | n | 2 | 0 | 2 | 6 | { |

같은 글자가 같은 6-column symbol로 반복됨을 확인할 수 있다.

- `h` → symbol A (0번째, 5번째 동일)
- `2` → symbol H (9번째, 11번째 동일)
- `t`, `e` 등도 뒤에서 동일 symbol로 반복

전체 symbol sequence:

```
A B C D E A F C G H I H J K E A F L M N C E E F O L O F P F B N Q L R E Q L Q F C O F E L E A O C S T A L Q C S G U
```

Known mapping을 적용하고 반복되는 symbol을 문맥에 맞춰 채우면 최종 문자열을 복원할 수 있다.

```
hacktheon2026{the_plotter_reveals_its_secret_through_sound}
```
